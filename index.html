Notes

                    
This MVP is intentionally practical rather than exhaustive.

                

                

                    
    
                        
  • It handles the most common RFC 4515 filter forms used in many real systems.
    • 
                        
  • It rejects raw mandatory-escape characters inside assertion values unless properly escaped, while still allowing * for equality-based substring and presence filters.
    • 
                        
  • It does not implement extensible match syntax such as (cn:caseExactMatch:=John).
    • 
                        
  • Attribute validation is intentionally simplified for an MVP.
    • 
                        
  • The page runs entirely in the browser.
    •
